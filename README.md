# Task Tracker API

Сервис управления задачами с ролевой моделью, JWT-аутентификацией и системой фоновых почтовых уведомлений.

## 1. Стек технологий
- Язык программирования: Java 21 (LTS)
- Серверный фреймворк: Spring Boot 4.0.6
- СУБД: PostgreSQL 16 (контейнер)
- Инструмент миграций: Liquibase 4.20
- Почтовый сервер: Mailpit (SMTP-заглушка для тестирования)
- Сборщик проектов: Gradle 8
- Тестирование: JUnit 5, Mockito

## 2. Сетевая конфигурация и порты
Для исключения конфликтов с локальными СУБД PostgreSQL на машинах разработчиков, внешние порты инфраструктуры переопределены:
- `9090` — Клиентский интерфейс (Frontend SPA) и Backend API.
- `5434` — Внешний порт СУБД PostgreSQL (внутренний порт Docker-сети: `5432`).
- `8025` — Веб-интерфейс почтового клиента Mailpit (для просмотра писем).
- `1025` — SMTP-порт Mailpit (для приема писем от приложения).

## 3. Схема данных и миграции
Структура таблиц полностью контролируется инструментом Liquibase (директория `src/main/resources/db/changelog/`). 
Ручное создание таблиц `users` и `tasks` в базе данных не требуется. При успешном старте Spring-контекста Liquibase автоматически проверит состояние базы данных и выполнит накат необходимых таблиц.

## 4. Инструкция по запуску проекта

Проект поддерживает два альтернативных сценария развертывания.

### Сценарий А: Полный запуск в Docker-контейнерах
Данный сценарий собирает образ приложения и запускает весь стек сервисов одной командой.
1. Убедитесь, что Docker Desktop запущен.
2. Откройте терминал в корневой директории проекта и выполните сборку:
   docker compose up --build -d
3. Дождитесь успешного запуска контейнера `task_backend`.

### Сценарий Б: Локальный запуск (Разработка в IDE)
Данный сценарий запускает инфраструктуру в Docker, позволяя выполнять отладку Java-кода непосредственно из среды разработки.
1. В корневой директории проекта запустите только базу и почту:
   docker compose up postgres mailpit -d
2. Запустите метод `main` в классе `TasktrackerApplication.java` из вашей IDE.
   Приложение применит конфигурацию из `application.properties` и установит соединение с СУБД через порт `localhost:5434`.

## 5. Точки входа и начало тестирования
После развертывания системы перейдите по адресам:
- Интерфейс приложения: http://localhost:9090
- Интерфейс почты (Mailpit): http://localhost:8025

**Внимание:** База данных инициализируется пустой. Предустановленных учетных записей нет. 
Для начала работы:
1. Перейдите на страницу http://localhost:9090.
2. Заполните поля Email/Пароль и нажмите **«Зарегистрироваться»**.
3. Система создаст пользователя, выполнит накат таблиц и сгенерирует JWT-токен для авторизации последующих запросов.
4. Исходящее приветственное письмо будет доставлено в Mailpit (http://localhost:8025).

## 6. Запуск автоматических тестов
Для выполнения набора модульных и интеграционных тестов используйте команды:
В терминале запустить бд в фоновом режиме:

docker compose up postgres -d

Потом запуск тестов:

./gradlew test
